后端脚手架项目

基于Spring Boot的DDD架构RBAC权限模型后端系统

领域驱动设计 (DDD)

采用领域驱动设计架构，将业务逻辑与技术实现分离，提高代码的可维护性和可扩展性。

基于角色的访问控制 (RBAC)

实现了完整的RBAC权限模型，支持用户、角色、权限的灵活配置和管理。

RESTful API

提供标准的RESTful API接口，支持JSON格式的数据交换，便于前后端分离开发。

安全性

集成Spring Security和JWT令牌认证，保障系统的安全性。

数据持久化

使用Spring Data JPA简化数据访问层的开发，支持多种数据库。
